# AI Context

This repository stores design context, not executable implementation.

Agents should use this repo to understand project intent, architecture, decisions, questions, and links to implementation repositories. Do not infer that code belongs here unless a document explicitly says a file is a non-executable artifact.

## Rules for Agents

- Preserve the distinction between design and implementation.
- Prefer editing or adding markdown context files over creating executable code.
- Link to implementation repositories instead of copying source code here.
- Keep project-specific notes under `projects/<project>/`.
- Keep reusable concepts under `shared/`.
- Use decision records for meaningful direction changes.
- Keep speculation labeled as speculation.
- Preserve provenance when importing context from chats or other tools.

## Context Portability

Write docs so they can be read by humans and by agents in Claude, Codex, ChatGPT, Cursor, and similar tools. Favor clear headings, explicit assumptions, dated decisions, and links over implicit conversational memory.
