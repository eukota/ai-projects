# Testing

How candidates validate solutions with AI.

Observe during the Testing phase. Look for critical evaluation of tests and strategic test coverage.

## Observable Signals

### Critical Evaluation of Test Inputs
*Does the candidate review test quality critically?*

**Low:**
- Accepts AI-generated tests as-is
- Doesn't review test cases
- Doesn't check input coverage

**Medium:**
- Reviews tests
- Spots some gaps (e.g., missing input types)
- Makes a few requests to improve

**High:**
- Carefully evaluates test inputs
- Identifies gaps in coverage (special characters, edge cases, boundaries)
- Makes deliberate improvements to test suite

---

### Active Direction
*Does the candidate guide test strategy?*

**Low:**
- Doesn't specify test types
- Accepts what AI generates
- No iteration

**Medium:**
- Requests specific test types (unit, integration, etc.)
- Makes some adjustments
- Focuses on main flows

**High:**
- Explicitly requests test strategy (unit, integration, etc.)
- Discusses coverage approach
- Adds edge cases based on domain knowledge

---

### Adding Value
*Does the candidate add their own test cases?*

**Low:**
- Only uses AI-generated tests
- Doesn't add anything

**Medium:**
- Adds one or two edge cases
- Contributes to test suite

**High:**
- Adds multiple meaningful edge cases
- Shows domain knowledge in test selection
- Demonstrates testing strategy thinking

---

### Articulation
*Does the candidate explain their testing decisions?*

**Low:**
- Silent about test choices
- No explanation

**Medium:**
- Explains some test decisions
- Discusses coverage concerns

**High:**
- Clearly articulates testing strategy
- Explains why each test matters
- Discusses tradeoffs (speed vs. coverage)

---

## Scoring

**Low:** Doesn't review tests, accepts blindly, no additions  
**Medium:** Reviews tests, makes requests, adds some cases  
**High:** Critical evaluation, strategic coverage, adds meaningful tests, explains thinking
