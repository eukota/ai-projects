# Implementation

How candidates implement code with AI.

Observe during the Implementation phase. Look for active direction, critical thinking, and proper use of AI controls.

## Observable Signals

### Active Direction & Critical Thinking
*Does the candidate review and direct code generation?*

**Low:**
- Blindly accepts AI-generated code
- Doesn't read the output
- Says "looks good" without inspection

**Medium:**
- Reads the code
- Spots some issues or makes minor requests
- Makes at least one change

**High:**
- Carefully reviews code structure
- Identifies missing pieces or improvements
- Makes deliberate changes showing understanding

---

### Articulation
*Does the candidate explain what they're evaluating?*

**Low:**
- Silent while reviewing code
- No explanation of findings

**Medium:**
- Explains some of their observations
- Talks through what they're checking

**High:**
- Clearly articulates what they see
- Explains why changes matter
- Demonstrates code understanding

---

### Strategic Constraints
*Do they intelligently trade off quality for time?*

**Low:**
- Doesn't mention time constraints
- Tries to build everything
- Runs out of time

**Medium:**
- Mentions MVP or time constraint
- Skips some features
- Executes mostly in time

**High:**
- Explicitly tells AI the constraints (time, MVP, in-memory DB, etc.)
- Strategically skips quality-of-life features
- Completes clean implementation on schedule

---

### Collaboration
*Do they iterate and refine with AI?*

**Low:**
- Accepts first output
- Doesn't iterate

**Medium:**
- Makes a few requests for changes
- Some back-and-forth with AI

**High:**
- Iterates to shape code
- Uses AI as thinking partner
- Refines implementation collaboratively

---

## Scoring

**Low:** Doesn't read code, blindly accepts, no iteration  
**Medium:** Reviews code, makes some changes, basic constraints  
**High:** Critical review, strategic constraints, active direction, collaborative iteration
