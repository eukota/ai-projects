# Level 4: Consumer

You use AI-powered autocomplete and suggestions in your IDE. AI feels like fancy autocomplete—you might not even think of it as "using AI." It's just there.

## What This Stage Looks Like

- GitHub Copilot, Cursor, or similar installed as IDE extension
- You accept or reject AI suggestions as you type
- AI suggests completions based on context
- You're observing patterns in what works and what doesn't

## Skills at This Stage

**Accepting suggestions**
- Tab to accept, escape to reject
- Building muscle memory for quick decisions

**Triggering completions**
- Knowing when AI will suggest (e.g., after a function signature, within a loop)
- Understanding what context triggers useful completions

**Basic context awareness**
- Understanding that AI uses surrounding code to make suggestions
- Noticing how comments and variable names influence suggestions

## Tools

- GitHub Copilot
- Cursor
- Codeium
- Amazon CodeWhisperer
- IDE-native plugins (JetBrains, VS Code, etc.)

## Exercises

**1. Observe the suggestions**
- Over a week of normal coding, track: How many suggestions does AI offer? How many do you accept? Which were helpful? Which were wrong? What patterns triggered good suggestions?

**2. Comment-driven development**
- Write a clear comment describing what you want to do
- Let AI complete the code
- Example: `// Parse the JSON response and extract the user ID` → AI completes the implementation

**3. Learn the shortcuts**
- Bind the accept/reject keys to muscle memory
- Experiment with keyboard vs. mouse interaction

## Common Mistakes

- Accepting suggestions without reading them
- Fighting the AI (turning it off because early suggestions were bad)
- Ignoring it entirely (not developing intuition for when it helps)

## Signs You're Ready for the Next Stage

- You instinctively know when AI will help versus hinder
- You've developed a feel for good versus bad suggestions
- You want to ask AI questions, not just accept completions
- You've thought: "I wish I could just tell it what I want"
