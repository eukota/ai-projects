# Level 5: Conversational

You have conversations with AI to understand, debug, and design. AI is a knowledgeable colleague you can ask anything. You copy code in and out of chat. You're learning how to ask good questions and iterate on answers.

## What This Stage Looks Like

- Using ChatGPT Cloud, Claude, or IDE chat features
- Asking questions: "Why is this failing?" "How should I structure this?"
- Pasting code, errors, and context into chat
- Treating AI as a colleague, not just a code generator
- Refining answers through follow-up questions

## Skills at This Stage

**Asking good questions**
- Clear, specific, with relevant context

**Providing context**
- Sharing relevant code, error messages, and goals
- Explaining what you're trying to accomplish

**Iterating on answers**
- Following up and refining responses
- Drilling down into details
- **Critical: Learning to follow-up effectively instead of accepting answers blindly**

**Evaluating responses**
- Knowing when AI is wrong or hallucinating
- Fact-checking important claims

## Tools

- Claude.ai (web)
- ChatGPT / ChatGPT Plus
- Cursor Chat
- GitHub Copilot Chat
- IDE-integrated chat features

## Art of Prompting

**Bad prompt:** "This doesn't work"

**Good prompt:** "I'm trying to parse JSON from an API response in Go. I'm getting 'unexpected end of JSON input'. Here's my code: [code]. What's wrong?"

**Principles:**
- Always include: what you're trying to do, what's happening, what you expected, relevant code/errors
- Provide context so AI can understand the full picture
- Follow-up effectively—this is where most engineers fail (accepting answers without verification or refinement)

## Exercises

**1. Explain this code**
- Paste code into chat and ask AI to explain what it does at different levels of detail

**2. Debug with AI**
- Share an error and relevant code; work through the fix interactively

**3. Design discussion**
- Talk through a design decision or architecture question with AI

**4. Learn to spot hallucinations**
- Ask AI something you know the answer to; compare and calibrate your trust

## Common Mistakes

- Trusting without verifying (especially for facts, APIs, syntax)
- Lazy context (not providing enough information)
- Not iterating (accepting the first answer instead of refining)
- Copy-paste coding (using code without understanding it)

## Signs You're Ready for the Next Stage

- You can get useful answers consistently
- You know when to trust AI and when to verify
- You've thought: "I wish AI could just make this change for me"
- Copy-pasting code between chat and IDE feels tedious
