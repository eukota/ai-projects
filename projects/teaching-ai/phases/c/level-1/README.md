# Level 1: Foundations
## Zero or Near-Zero AI

You're at the beginning of your AI-assisted coding journey. Right now, you're mostly writing code the traditional way—maybe using code completions occasionally, or asking ChatGPT a question now and then. AI is a tool you reach for, not a partner in your workflow.

### Where You Are

Pre-agent. You use AI sporadically, on your own terms, without systematic integration into your development process.

---

## Understanding This Stage

### Philosophy

At this stage, you're establishing the foundation for what comes next. The goal isn't to use AI everywhere; it's to understand what AI can actually help with, build comfort with the technology, and start noticing patterns in your work where AI might accelerate you.

This is the "skeptic" stage—and that's healthy. You're testing boundaries, learning what's possible, and being intentional about when and how you use AI.

### How It Feels

- Traditional development with occasional AI help
- AI is external (separate chat windows, not in your IDE)
- You're the decision-maker; AI provides input
- Lower velocity, but high control and understanding

---

## Getting Work Done

### Typical Workflow

- Write code directly in your IDE or editor
- Occasionally paste a problem into ChatGPT or Claude for explanation or brainstorming
- Use code completion features (GitHub Copilot, etc.) but don't rely on them heavily
- Manually verify all outputs; trust is low

### When AI Helps

- Debugging cryptic error messages
- Generating boilerplate (tests, configs, schemas)
- Explaining existing code
- Brainstorming architecture before coding

### When AI Hurts

- Using it for core algorithm design without understanding the output
- Treating AI-generated code as tested code
- Asking vague questions and accepting vague answers

### Patterns & Anti-Patterns

#### What Works

- Ask specific questions with context (paste code, not just descriptions)
- Verify AI output by running it yourself
- Use AI to accelerate research, not replace thinking
- Build a personal knowledge base of "what AI is good for"

#### What Breaks

- Expecting AI to understand your entire codebase from a sentence
- Copy-pasting large blocks of code without reading them
- Trusting AI's confidence as a proxy for correctness
- Assuming one answer is the only answer

---

## Growing Here

### Learning Exercises

1. **Pair Programming with Chat:** Pick a small feature. Ask ChatGPT to help you design it (architecture, pseudocode). Compare its suggestion to your own. What's missing? What's clever?

2. **Boilerplate Generation:** Generate test boilerplate with AI, then run the tests. What did it assume? What did it miss?

3. **Error Message Decoding:** When you hit an error, try explaining it to Claude before searching Stack Overflow. Compare the explanations.

4. **Code Review with AI:** Paste a small piece of your code into ChatGPT and ask for a code review. How accurate is the feedback? Where does it miss context?

### Ready to Level Up?

You're ready to move to **Level 2: Emerging** when:
- You've used ChatGPT/Claude enough to feel comfortable with AI-generated code
- You understand that AI hallucinations are real and you can spot them
- You're curious about using an agent *inside* your IDE rather than in a separate window
- You want to explore what an AI agent can do with access to your filesystem and tools (but with your permission)
