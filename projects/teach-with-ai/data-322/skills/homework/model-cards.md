# Homework Skill: Model Cards

## TODO

Model card structure: model details, intended use, factors (relevant demographic and environmental), metrics, evaluation data, training data, ethical considerations, caveats and recommendations. Common deficiencies: accuracy reported without confidence intervals or dataset size context, no mention of subgroup performance, intended use section so vague it doesn't constrain misuse, training data section missing provenance. Walk a student through the dean's test drive scenario: hospital readmission model card with overconfident accuracy claim, missing demographic performance breakdown, no documentation of training data recency.

## Quiz Prep

The AI will quiz you on these questions in Socratic mode before the in-class test — it will ask you to answer first, then give feedback. Memorizing definitions won't be enough; you need to be able to reason through these in plain language.

1. **A model card's Metrics section says "accuracy: 0.91." What is missing, and why can't a stakeholder evaluate that number without more context?**

2. **What is the purpose of the Intended Use section, and what makes an intended use statement too vague to be useful? Give an example of a weak statement and what a stronger one would include.**

3. **Why does a model card need both a Training Data section and an Evaluation Data section? What question does each one answer?**

4. **Your model card's Ethical Considerations section says "this model could be used inappropriately." Why is that insufficient, and what would a concrete, actionable ethical consideration look like?**

5. **A model was trained on hospital records from 2016–2019 and deployed in 2027. Where in the model card does this matter, and what specific risk should the card flag?**
