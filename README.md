# AI Projects

Durable home for AI project ideas, architecture notes, planning context, and design evolution.

This repository is not an implementation monorepo.

## Core Rule

Design lives here.
Code lives elsewhere.
Context links them.

Executable software must live in separate repositories and be referenced from the relevant project documents.

## Goals

- Preserve project ideas cleanly.
- Reduce scattered notes across tools.
- Make context portable across Claude, Codex, ChatGPT, Cursor, and other agents.
- Keep implementation repositories clean and focused.
- Provide a reusable project structure for future ideas.

## Repository Map

- `AI_CONTEXT.md` - How agents should understand and use this repo.
- `AI_PRINCIPLES.md` - Operating principles for project context and agent collaboration.
- `AI_MODELS.md` - Notes about useful models, tradeoffs, and agent roles.
- `AI_WORKFLOW.md` - Workflow for adding, refining, and linking project context.
- `_templates/` - Copyable templates for project docs and records.
- `projects/` - One folder per project idea.
- `shared/` - Cross-project glossary, principles, open questions, and repo map.
